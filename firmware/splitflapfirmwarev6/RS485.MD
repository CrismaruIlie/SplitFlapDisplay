# RS485 Command Reference

Protocol implemented by `splitflapfirmwarev6.ino` for split-flap display modules on a shared RS485 bus.

## Physical Layer

| Setting | Value |
|---------|-------|
| Baud rate | 9600 |
| Data format | 8N1 (Arduino `SoftwareSerial` default) |
| Direction control | DE pin on GPIO 2 (HIGH = transmit, LOW = receive) |
| RX / TX | GPIO 3 / GPIO 1 |

Modules share one bus. Only the module whose ID matches the command (or a broadcast wildcard) executes it.

## Message Format

All commands follow this pattern:

```
m<ID><command>[parameters]
```

| Field | Description |
|-------|-------------|
| `m` | Fixed start byte |
| `<ID>` | Two-character module address (see below) |
| `<command>` | Single-letter command code |
| `[parameters]` | Optional; format depends on command |

### Module Addressing

The two ID characters are the module's decimal ID, zero-padded to two digits:

| Module ID | Address bytes |
|-----------|---------------|
| 0 | `00` |
| 1 | `01` |
| 9 | `09` |
| 10 | `10` |
| 44 | `44` |

**Broadcast / wildcard:** use `*` for either or both ID characters (e.g. `m**h` homes every module that accepts the command).

Example — move module 3 to letter `A`:

```
m03-A
```

## Numeric Parameters

Commands that take a number (`o`, `t`, `s`, `g`, `i`, `a`, and the position half of `w`) accept one or more ASCII digits (`0`–`9`).

The number ends when:

1. A non-digit character is received, or
2. No serial data arrives for **50 ms** (timeout flush in the main loop)

Either terminator completes the command.

## Character Set

The move command (`-`) accepts any character from this reel (64 positions, index 0–63):

```
 ABCDEFGHIJKLMNOPQRSTUVWXYZ0123456789!@#$&()-+=;q:%'.,/?*roygbpw
```

Index 0 is a space. Characters not in this string are ignored.

## Commands

### `-` — Move to character

```
m<ID>-<char>
```

Rotates the drum to display `<char>`. Always moves forward (backward in stepper terms) along the shortest path around the reel.

- If the module has no known position (`currentFlapIndex == -1`), it homes first.
- If already showing the target character, no movement occurs.
- Uses a per-character EEPROM map when available; otherwise falls back to even spacing around the reel.

**Example:** `m01-H` → module 1 shows `H`.

**Response:** none.

---

### `h` — Home

```
m<ID>h
```

Finds the Hall sensor, backs off by the stored home offset, and sets logical position to step 0 / character index 0.

**Example:** `m01h`

**Response:** none.

---

### `c` — Calibrate (measure reel length)

```
m<ID>c
```

Automatically measures how many steps constitute one full revolution using the Hall sensor, saves the result as `totalStepsPerRev`, homes, and persists state.

**Example:** `m01c`

**Response:**

```
m<ID>:<measuredSteps>\n
```

Example: `m01:4096\n`

The module waits 50 ms before transmitting so the host adapter can switch to receive mode.

---

### `o` — Set home offset

```
m<ID>o<steps>
```

Sets `stepsFromHallToZero` — the number of steps from the Hall sensor edge to the logical zero (character index 0) position. Saved to EEPROM.

**Example:** `m01o2832`

**Response:** none.

---

### `t` — Set total steps per revolution

```
m<ID>t<steps>
```

Sets `totalStepsPerRev` manually (normally set by `c` calibration). Saved to EEPROM.

**Example:** `m01t4096`

**Response:** none.

---

### `s` — Step backward (adjust offset)

```
m<ID>s<steps>
```

Moves the motor backward by `<steps>` steps, then **adds** that count to `stepsFromHallToZero` and saves the new home offset. Used for fine-tuning alignment without a full home cycle.

**Example:** `m01s10` → move 10 steps backward and increase home offset by 10.

**Response:** none.

---

### `g` — Go to absolute step position

```
m<ID>g<step>
```

Moves to absolute step `<step>` (0 … `totalStepsPerRev - 1`), wrapping forward if needed. Sets the character index to unknown (`-2`) because the position may not align with a mapped character.

**Example:** `m01g2048`

**Response:** none.

---

### `w` — Write character map entry

```
m<ID>w<index>:<position>
```

Stores a calibrated step position for character index `<index>` (0–63) in EEPROM. Used after calibration to map each character to an exact step count.

| Part | Description |
|------|-------------|
| `<index>` | Character index (see character set above) |
| `<position>` | Step value when that character is centered in the window |

**Example:** `m01w0:0` → index 0 (space) is at step 0.  
**Example:** `m01w8:512` → index 8 (`G`) is at step 512.

**Response:** none.

---

### `i` — Set module ID

```
m<ID>i<newId>
```

Assigns a new module ID (0–99), saves it to EEPROM, and updates the address the module listens for on subsequent commands.

**Example:** `m01i05` → re-address module 1 as module 5 (must currently be addressed as `01`).

**Response:** none.

---

### `a` — Set auto-home on boot

```
m<ID>a<flag>
```

| `<flag>` | Behavior |
|----------|----------|
| `1` | On power-up, home automatically and save position after moves |
| `0` | On power-up, restore last saved step position and character index |

Saved to EEPROM.

**Example:** `m01a0` → disable auto-home.

**Response:** none.

---

### `d` — Dump EEPROM / status

```
m<ID>d
```

Transmits current calibration data and the character map.

**Example:** `m01d`

**Response:**

```
m<ID>d:<homeOffset>:<totalSteps>:<map>\n
```

| Field | Description |
|-------|-------------|
| `<homeOffset>` | Current `stepsFromHallToZero` |
| `<totalSteps>` | Current `totalStepsPerRev` |
| `<map>` | Comma-separated `index=position` pairs for mapped characters (empty entries omitted) |

Example:

```
m01d:2832:4096:0=0,8=512,16=1024
```

The module waits 50 ms before transmitting so the host adapter can switch to receive mode.

---

### `e` — Erase character map

```
m<ID>e
```

Clears all 64 character map entries in EEPROM (sets each to `0xFFFF`). After erase, moves use evenly spaced default positions until the map is rewritten with `w`.

**Example:** `m01e`

**Response:** none.

---

## Quick Reference

| Command | Format | EEPROM | Response |
|---------|--------|--------|----------|
| Move to char | `m<ID>-<char>` | — | — |
| Home | `m<ID>h` | saves state | — |
| Calibrate | `m<ID>c` | total steps | `m<ID>:<steps>\n` |
| Set home offset | `m<ID>o<steps>` | yes | — |
| Set reel length | `m<ID>t<steps>` | yes | — |
| Step + adjust offset | `m<ID>s<steps>` | yes | — |
| Go to step | `m<ID>g<step>` | saves state | — |
| Write map entry | `m<ID>w<idx>:<pos>` | yes | — |
| Set module ID | `m<ID>i<id>` | yes | — |
| Auto-home flag | `m<ID>a<0\|1>` | yes | — |
| Dump status | `m<ID>d` | — | `m<ID>d:...\n` |
| Erase map | `m<ID>e` | yes | — |

## Typical Workflow

1. **Address modules** — flash firmware with a unique `HARDCODED_ID`, or assign at runtime with `i`.
2. **Calibrate reel** — send `c`; read back measured step count.
3. **Home** — send `h`.
4. **Fine-tune offset** — use `s` or `o` if the zero position needs adjustment.
5. **Map characters** — write each index/position pair with `w` (or verify with `d`).
6. **Display** — send `-<char>` to show characters.

## Notes

- On boot, modules stagger startup by `moduleId × 150 ms` to reduce bus collisions.
- When auto-home is enabled, position and character index are not written to EEPROM on every move; when disabled, they are restored after reboot.
- Responses from `c` and `d` drive DE HIGH for ~100 ms; hosts should allow time to switch from transmit to receive.
